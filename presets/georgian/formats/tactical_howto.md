# Tactical How-To Format

Use this format when the post hands a practitioner a specific applied pattern — eval design, RAG architecture, fine-tuning tradeoffs, GTM sequencing.

- Length: 800–1,200 words.
- Open with the outcome and the practitioner who needs it. Name the prerequisite stack or stage.
- Walk the pattern in order. Each step: the action, the reason it matters, and the failure mode if skipped.
- Cite numbers, benchmarks, or customer-observed behaviour wherever a claim implies them.
- Surface the tradeoffs honestly — latency, cost, infra burden, evaluation overhead.
- Close with a "you're done when..." signal and the next decision the reader should now be ready to make.
